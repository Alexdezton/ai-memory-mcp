# PROJECT STRUCTURE & FILE ORGANIZATION GUIDELINES

Настоящий регламент описывает профессиональные стандарты организации файлов проекта. Агент **ОБЯЗАН** следовать этой структуре при создании и редактировании любых файлов.

---

## 📂 Базовая структура папок проекта

Для предотвращения хаоса в корневом каталоге проекта **запрещено** размещать файлы исходного кода (`.css`, `.js`, `.ts`, `.jsx`, `.tsx`, `.scss`) в корне. Корень проекта предназначен исключительно для файлов конфигурации и единой точки входа (`index.html`).

Используйте следующие стандарты распределения файлов:

### 1. Простые веб-сайты (HTML/CSS/JS)
```
├── AI_MEMORY/           # Проектная память (документация)
├── css/                 # Стили интерфейса
│   ├── variables.css    # CSS-переменные (цвета, шрифты)
│   ├── layout.css       # Сетки, разметка и позиционирование
│   └── components.css   # Стили отдельных элементов (кнопки, карточки)
├── js/                  # Логика и данные
│   ├── data.js          # Локальные данные и заглушки
│   └── script.js        # Управляющий скрипт страницы
├── index.html           # Входная страница (единственный разрешенный файл в корне)
├── mcp_ai_memory_config.json
└── package.json
```

### 2. Современные фреймворки (React / Vite / Next.js)
```
├── AI_MEMORY/
├── src/                 # Весь исходный код приложения
│   ├── components/      # Переиспользуемые UI-компоненты
│   ├── styles/          # Глобальные или модульные CSS-стили
│   ├── main.jsx         # Точка входа React
│   └── App.jsx          # Корневой компонент
├── public/              # Статические файлы (картинки, шрифты)
├── mcp_ai_memory_config.json
└── package.json
```

## 🚨 Ошибки верификации

При попытке записать файлы исходного кода напрямую в корень проекта, инструмент верификации `verify_write` заблокирует выполнение и вернет ошибку `ROOT_SOURCE_FORBIDDEN`.

**Что делать при возникновении ошибки:**
1. Создайте нужную структуру папок (например, `css/` и `js/`).
2. Перенесите исходный код туда.
3. Обновите пути подключения стилей и скриптов в `index.html`.
4. Запустите `verify_write` для новых путей.
