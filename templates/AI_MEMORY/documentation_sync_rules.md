# DOCUMENTATION SYNC RULES

**Purpose:** Автоматическое обновление документации сессии.

---

## Основные правила:
1. Документация обновляется по команде `GIT COMMIT` или `EOS`.
2. Агент определяет измененные файлы и заносит их в `session_log.md` (раздел "Files changed").
3. Подробная карта триггеров (какой файл когда обновлять) описана в регламенте: [DOCUMENTATION_SYNC.md](ai_agent_guidelines/workflow/DOCUMENTATION_SYNC.md).
