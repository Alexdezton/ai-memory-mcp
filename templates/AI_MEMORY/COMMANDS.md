# Команды для пользователя (User Commands Reference)

Этот файл содержит справочник всех специальных команд, которые можно использовать в чате с агентом.

---

## ⚡ Шпаргалка команд (Quick Reference)

**Копируй и вставляй в чат:**

- `LOADER` — загрузить AI_MEMORY контекст
- `GIT COMMIT` — обновить документацию + git commit (см. documentation_sync_rules.md)
- `EOS` — сохранить checkpoint сессии и создать summary
- `SET PROJECT_STATE [development|release]` — переключить режим поведения агента

---

## 📋 Подробное описание команд

### `LOADER`
**Описание:** Загрузить МИНИМУМ контекста для старта сессии (3 файла).

**Когда использовать:**
- В начале каждой новой сессии

**Что делает агент:**
1. Читает `AI_MEMORY/LOADER.md` и следует протоколу
2. Загружает ТОЛЬКО 3 файла:
   - `core_rules.md` — правила, PROJECT_STATE
   - `tasks.md` — задачи, ACTIVE_TASK, NEXT_STEP
   - `checkpoint-latest.json` — состояние (если есть)
3. Выводит: "AI_MEMORY LOADED" + статус (ACTIVE_TASK, NEXT_STEP, BLOCKERS, PROJECT_STATE)
4. Остальные файлы читает ПО МЕРЕ НЕОБХОДИМОСТИ (см. agent.md)

---

### `GIT COMMIT`
**Описание:** Обновить документацию по изменениям сессии + сделать git commit.

**Когда использовать:**
- Когда все задачи сессии выполнены и пользователь готов зафиксировать

**Что делает агент:**
1. Анализирует ВСЕ изменения текущей сессии
2. Обновляет документацию по карте триггеров (documentation_sync_rules.md)
3. Делает `git add . && git commit -m "[описание]"`
4. Сообщает результат

---

### `MEETING_END`
**Описание:** Сохранить протокол совещания в `brainstorm/`.

**Когда использовать:**
- После завершения обсуждения архитектурных решений

**Что делает агент:**
1. Проверяет `brainstorm/` — есть ли файл с такой темой?
2. Если нет — создаёт `AI_MEMORY/brainstorm/[NN]_[topic].md` (01_, 02_, ...)
3. Если есть — добавляет запись в конец существующего файла

**Когда использовать:**
- Когда агент делает слишком много вызовов инструментов
- Когда агент забыл про параллельные вызовы

**Что делает агент:**
- Читает протоколы эффективной работы
- Применяет правила: read целиком, параллельные вызовы, grep вместо search_codebase

---

### `EOS` (End of Session)
**Описание:** Сохранить прогресс сессии и создать чекпоинт для следующей сессии.

**Когда использовать:**
- ✅ В конце рабочей сессии
- ✅ После завершения крупной фичи/задачи
- ✅ Перед переключением на другую задачу
- ✅ Когда контекст приближается к лимиту

**Что делает агент:**
1. **Создаёт summary (5 пунктов):**
   - Что сделано
   - Что изменено (файлы)
   - Решения (добавляет в decisions.md)
   - Блокеры
   - NEXT_STEP (конкретный следующий шаг)

2. **Обновляет AI_MEMORY файлы:**
   - Добавляет summary в `session_log.md`
   - Обновляет статусы в `tasks.md`
   - Добавляет новые решения в `decisions.md` (если были)

3. **Создаёт checkpoint:**
   - Создаёт/обновляет `checkpoint-latest.json`

4. **Выводит подтверждение:**
   ```
   CHECKPOINT SAVED: {ACTIVE_TASK, NEXT_STEP, BLOCKERS}
   ```

---

### `SET PROJECT_STATE`
**Описание:** Переключить режим поведения агента.

**Значения:**
- `SET PROJECT_STATE development` — агент предлагает идеи и варианты (по умолчанию)
- `SET PROJECT_STATE release` — строгое выполнение без предложений
